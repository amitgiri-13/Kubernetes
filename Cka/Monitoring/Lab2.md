# Kubernetes Application Logs Lab — Notes

# Objective

Learn how to:

* Inspect application logs
* Troubleshoot application issues
* Identify errors from logs
* Work with multi-container pod logs

---

# Viewing Running Pods

Check deployed pods:

```bash id="j8m1rp"
kubectl get pods
```

Verify:

* Pod status
* Running applications

---

# Viewing Application Logs

Use:

```bash id="m2q7dx"
kubectl logs <pod-name>
```

Example:

```bash id="k4n8tb"
kubectl logs webapp
```

This displays application logs generated by the container.


# Problem Encountered

Running:

```bash id="g9y4le"

```

fails because:

* The pod has multiple containers.
* Kubernetes requires specifying the container name.

# Multi-Container Pod Logging

## Important Concept

If a pod has multiple containers:

```bash id="z8v2cn"
kubectl logs <pod-name>
```

may fail.

You must specify:

```bash id="n1d6ps"
kubectl logs <pod-name> <container-name>
```

# Key Commands Summary

## Get Pods

```bash id="h7r5bk"

```

---

## View Pod Logs

```bash id="t4u9jc"
kubectl logs <pod-name>
```

---

```bash id="r2x8fq"

```

---

```bash id="c5m1ye"

```

# Troubleshooting Workflow

```text id="b4k7wd"
User Issue
    ↓
Check Pod Status
    ↓
Inspect Logs
    ↓
Identify Warnings/Errors
    ↓
Find Root Cause
```

---

# Key Takeaways

* Application logs are critical for troubleshooting.
* `kubectl logs` is used to inspect container logs.
* Multi-container pods require specifying container names.
* Logs help identify:

  * Login failures
  * Application errors
  * Inventory issues
  * Runtime problems
* Warning and error logs are important during debugging.
* Kubernetes logging is straightforward and effective for application troubleshooting.
