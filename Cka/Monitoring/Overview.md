# Kubernetes Logging and Monitoring — Notes

## Overview

This section covers the **logging** and **monitoring** capabilities available in Kubernetes.

Main focus areas:

* Monitoring Kubernetes cluster components
* Monitoring deployed applications
* Viewing and managing logs

---

# Monitoring in Kubernetes

## What is Monitoring?

Monitoring helps track:

* Cluster health
* Resource usage
* Application performance
* System behavior

It allows administrators to identify:

* Failures
* Resource bottlenecks
* Performance issues

---

## Components to Monitor

### Cluster Components

Important Kubernetes components include:

* API Server
* Controller Manager
* Scheduler
* etcd
* Kubelet

### Application Components

Applications running inside:

* Pods
* Containers
* Deployments
* Services

---

## Monitoring Metrics

Common metrics monitored:

* CPU usage
* Memory usage
* Disk usage
* Network traffic
* Pod health
* Node status

---

## Common Monitoring Tools

### Metrics Server

Collects basic resource metrics such as:

* CPU
* Memory

Useful commands:

```bash
kubectl top node
kubectl top pod
```

---

### Prometheus

Popular monitoring and alerting system.

Features:

* Time-series metrics collection
* Alerting
* Dashboard integration

Often used with:

* Grafana

---

### Grafana

Visualization platform used to create dashboards for:

* Cluster metrics
* Application metrics
* Alerts

---

# Logging in Kubernetes

## What is Logging?

Logging records events and outputs generated by:

* Kubernetes components
* Applications
* Containers

Logs help troubleshoot issues and analyze system behavior.

---

## Viewing Logs

### View Pod Logs

```bash
kubectl logs <pod-name>
```

Example:

```bash
kubectl logs nginx-pod
```

---

### View Logs from Specific Container

```bash
kubectl logs <pod-name> -c <container-name>
```

---

### Stream Logs in Real Time

```bash
kubectl logs -f <pod-name>
```

`-f` means follow the logs continuously.

---

## Cluster Component Logs

Logs can also be collected from:

* kube-apiserver
* kube-scheduler
* kube-controller-manager
* kubelet

Depending on setup, logs may be available through:

* systemd journal
* container runtime logs
* centralized logging systems

---

# Centralized Logging Solutions

Common tools:

* Elasticsearch
* Fluentd
* Kibana (EFK Stack)
* Loki
* Promtail

Benefits:

* Centralized log storage
* Search and filtering
* Visualization
* Long-term retention


---

# Key Takeaways

* Monitoring tracks cluster and application health.
* Logging helps debug and troubleshoot issues.
* `kubectl logs` is used for viewing container logs.
* Metrics Server provides basic resource metrics.
* Prometheus and Grafana are widely used for monitoring.
* Centralized logging improves observability and troubleshooting.
